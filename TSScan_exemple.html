l'objectif de cet algo est de crer un espace de mtrique sur lequel appliquer la mthode dbscan. Aprs un tat de l'art j'ai eu l'ide d'utiliser le dynamic time warping pur notre espace de calcul.

Cet algorythme calcule une proximit entre 2 time sries en excluant les possibles dcalages temporels et dilatation temporelles. Il retourne la distance minimale entre 2 time series recalibres l'une sur l'autre et le chemin de recalibration.

On applique l'algorithme de Distance Base Clustering pour les raisons suivantes :

  • Nous ne connaissons pas le nombre de clusters dtecter ce qui limite les mthodes type K_means
  • Son fonctionnement sous l'ide : "dans un rayon epsilon je cherche au minimum n voisins pour initialiser un cluster et je continue de chercher les voisins dans un environnement eps de chaques poinst du cluster jusqu' converger"
  • Nos donnes peuvent stagner dans une position pendant un certain temps

J'ai aussi travaill sur une mthode de dynamic based averaging pour nous permettre de conserver un lment moyen chaques clusters. L'ide est de moyenniser tout les points associ une dilatation du temps calcule par le chemin de recalibration.

Pour cel je pars du mdod de chaques cluster (lment du cluster qui semble le plus proches de tout les autres) et je moyennise sur celui-ci. Ce choix s'explique par les possibles variations de taille entre les time series (pb de perte de donne, retard au dcollage, oubli d'teindre l'emetteur l'atrissage, pb de dtection) que le mdod semble moyenniser.
